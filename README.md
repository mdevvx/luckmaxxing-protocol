# Luckmaxxing Protocol Bot

A Discord bot that delivers an 8-day gamified luck-cultivation training program through interactive dialogue-based conversations. Users enroll with a unique code, receive a private training channel, and progress through daily lessons by clicking through a coach/student dialogue format.

---

## Features

- Private training channel per user (support-ticket style)
- Interactive dialogue — Gamblor responses are clickable buttons
- Intro + Day 1 delivered immediately on enrollment; Days 2–8 sent automatically every 24 hours
- Role assigned on enrollment, removed on unenrollment
- Single-use enrollment codes generated by admins
- Full activity logging to a configurable log channel
- Per-server toggle — bot can be enabled/disabled independently on every server
- Guards prevent `/setup` running before the server is configured
- Persistent views — buttons survive bot restarts

## Setup

### 1. Clone and install dependencies

```bash
pip install -r requirements.txt
```

### 2. Create your `.env` file

```env
DISCORD_BOT_TOKEN=your_bot_token_here
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your_service_role_key_here
LOG_LEVEL=INFO
```

> **Important:** Use the `service_role` key from Supabase → Settings → API, **not** the `anon` key.

### 3. Run the database schema

Open your Supabase project → SQL Editor → paste the contents of `supabase_schema.sql` → Run.

This creates 4 tables: `enrollments`, `guild_settings`, `daily_progress`, `enrollment_ids`.

### 4. Enable Discord intents

In the [Discord Developer Portal](https://discord.com/developers/applications):

- Bot → enable **Server Members Intent** and **Message Content Intent**

### 5. Sync slash commands

```bash
python sync_commands.py
```

Run this once after first launch or whenever commands change. Guild commands appear instantly; global commands take up to 1 hour.

### 6. Start the bot

```bash
python bot.py
```

## Server Setup (Admin — do this in order)

Once the bot is in your server, run these commands in order:

### Step 1 — Configure the server

```
/configure role:@Baby Gamblor category:#Training category: log_channel:#bot-logs
```

All three options are optional and can be set separately or together. Without a role and category, `/setup` will be blocked.

| Option        | Description                                                  |
| ------------- | ------------------------------------------------------------ |
| `role`        | Role assigned to users when they enroll                      |
| `category`    | Discord category where private training channels are created |
| `log_channel` | Channel where the bot posts all activity logs                |

### Step 2 — Post the enrollment message

```
/setup
```

Creates the `#luckmaxxing-protocol` channel (if it doesn't exist) and posts the enrollment embed with the Enroll button.

### Step 3 — Generate enrollment codes

```
/generateid count:10
```

Creates single-use codes to share with members. Each code can only be used once.

## User Enrollment Flow

1. User clicks **Enroll in Luckmaxxing Protocol** in `#luckmaxxing-protocol`
2. A modal appears — user enters their 5-character enrollment code
3. Bot verifies the code → assigns the configured role → creates a private `#luckmaxx-{username}` channel
4. Bot posts a pinned onboarding embed in the channel
5. Intro dialogue starts immediately, followed by Day 1
6. Days 2–8 are delivered automatically every 24 hours

## Commands

### Admin Commands

| Command               | Description                                             |
| --------------------- | ------------------------------------------------------- |
| `/configure`          | Set training category, onboarding role, and log channel |
| `/setup`              | Post the enrollment embed in `#luckmaxxing-protocol`    |
| `/toggle on\|off`     | Enable or disable the bot for this server               |
| `/generateid [count]` | Generate 1–50 enrollment codes                          |
| `/listids`            | View all unused enrollment codes                        |
| `/unenroll @user`     | Remove a user, delete their channel, remove their role  |
| `/globalstats`        | View enrollment statistics across all servers           |
| `/status`             | View bot status and current server configuration        |

### User Commands

| Command     | Description                                       |
| ----------- | ------------------------------------------------- |
| `/progress` | Check your current training day and enrollment ID |
| `/stats`    | View enrollment statistics for this server        |

### Owner Commands

| Command | Description                                            |
| ------- | ------------------------------------------------------ |
| `/sync` | Sync slash commands globally and to the current server |

## Activity Logs

When a log channel is configured, the bot posts embeds for the following events:

| Event                | Trigger                         |
| -------------------- | ------------------------------- |
| User Enrolled        | Successful enrollment           |
| Enrollment Failed    | Bad/used code, already enrolled |
| Role Assigned        | Role given on enrollment        |
| Role Removed         | Role taken on unenrollment      |
| Day Completed        | User finishes a day's dialogue  |
| Training Complete    | User finishes Day 8             |
| User Unenrolled      | Admin uses `/unenroll`          |
| Bot Enabled/Disabled | Admin uses `/toggle`            |
| IDs Generated        | Admin uses `/generateid`        |
| Day Delivered        | Daily task loop sends a day     |

## Adding Training Content

Open `content/training.py`. Each day is a list of `(speaker, message)` tuples. Speaker is either `"Intern"` (coach) or `"Gamblors"` (student response).

```python
DAY_2 = [
    ("Intern", "Welcome to Day 2..."),
    ("Gamblors", "Ready, Chief!"),
    ("Intern", "Today we cover..."),
    ("Gamblors", "Understood!"),
    # Add as many exchanges as needed
]
```

- `"Intern"` lines display in the embed and the user reads them
- `"Gamblors"` lines become the clickable button the user presses to advance
- The last item in the list triggers the **Complete** button

## Bot Disabled Behaviour

When the bot is disabled via `/toggle off`:

- All commands return an ephemeral "bot is disabled" message
- The enrollment button in the protocol channel also shows a disabled message
- The daily task loop skips guilds where the bot is disabled
- `/toggle`, `/sync`, and `/configure` still work so admins can always recover

## Database Tables

| Table            | Purpose                                                            |
| ---------------- | ------------------------------------------------------------------ |
| `enrollments`    | One row per user per guild — tracks day, channel ID, enrollment ID |
| `guild_settings` | Per-guild config — bot toggle, category, role, log channel         |
| `enrollment_ids` | Single-use enrollment codes with used/unused status                |
| `daily_progress` | Analytics log of every day completion event                        |

## Logs

File logs are written to `logs/bot.log` with rotation (10 MB max, 5 backups). Console output is emoji-safe for Windows terminals.
